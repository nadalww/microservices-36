源自SOA的理论体系，注重敏捷、持续交付、DevOps等自底向上的实践
微服务不是新语言，不解决开发的问题，它是一种架构模式和思想

服务拆分的策略可以先浅后深，先易后难
微服务架构的演进，涉及业务/开发/测试/部署运维，多角色全功能团队协作
微服务架构是DevOps后时代的一种演进式架构
微服务与模块/依赖的本质区别在于是否能被独立部署

分布式系统的复杂度是微服务演进的极大挑战，带宽、延迟、超时以及数据一致性是服务多后谨慎考虑的。
